# 交接（前端）：TTS 全段落（深度/问答/作者介绍）+ 播放进度条 + 409 修复

> 后端已扩展（staging，含迁移 n1k0）：音频端点覆盖**全部内容**——guide、深度模块、问答、作者介绍。
> 全部**懒生成**：用户点哪段才生成那段那语言（最省成本）；生成后落库，同语言用户共享秒回。
> 本文含 4 件前端工作：①播放进度条 ②各段播放按钮 ③**409"生成中"修复（用户实测 bug）** ④可选句级高亮。

## 端点（统一一个）

`GET /api/v1/museums/{slug}/objects/{qid}/audio?language={lang}&section={...}`

| section 取值 | 念什么 | 说明 |
|---|---|---|
| `guide` | 主讲解 | 已上线（现有"听讲解"） |
| `background` / `analysis` / `facts` / `significance` / `photographer` / `context` 等 | 该深度模块正文 | **新**：section = tabs 里的 `section_code` |
| `qa&qa_sort={N}` | **问+答连念**（闭眼听需上下文） | **新**：`sort` 来自 content 响应 `suggested_questions[].sort`（新加法字段） |
| `artist_bio` | 作者介绍 | **新**：**按作者共享**——莫奈 50 件作品同一份音频，第一个人生成后全馆秒回 |

响应/状态码（全 section 一致）：
- **200** `{"audio_url": "<R2 URL>"}` → 播放
- **409** `{reason: audio_generating}` → **正在生成，不是错误**（见 ③）
- **404** `{reason: no_published_text}` → 该段该语言文字未就绪 → 提示「内容生成后可听」
- **503** `{reason: tts_failed}` → 真失败 → 才显示「音频暂不可用，请重试」
- 422 `{reason: qa_sort_required}` → section=qa 忘传 qa_sort（开发期错误）

## ① 播放进度条 + 剩余时间（必做，纯前端）

音频加载后播放器原生有 `duration`/`position`——进度条 + 剩余时间（`-1:23`）零后端依赖。加到现有播放条（guide）和新的各段播放处。语速档位（0.75/1/1.5/2x，见 06-07 交接）与进度条共存：变速后剩余时间按 `remaining / rate` 显示。

## ② 各段播放按钮

- **深度模块**：每个 tab 内容区加「播放」（section=该 tab 的 `section_code`）
- **问答**：每条展开的问答加「播放」（section=qa&qa_sort=该条的 `sort`）
- **作者介绍**：作者卡 bio 处加「播放」（section=artist_bio）
- 同屏多个音频：播放新的先停旧的（单实例播放器）

## ③ 409 修复（用户实测 bug，重要）

**现象**：点「听讲解」等一会儿总弹「音频暂不可用，请重试」，多点几次才能播。
**根因**：首次点击后端在生成（3-8 秒）；期间用户再点 → 撞段级锁 → 后端返 **409（生成中）**——前端把 409 和 503 都当失败弹了提示。首次请求若客户端超时太短也会误报。

**修法**：
1. **409 ≠ 错误**：收到 409 → 保持转圈，**静默等 2 秒自动重试**（循环直到 200 或 503，上限 ~30 秒）
2. 音频请求超时 ≥ 30 秒（同步生成 3-8 秒 + 网络余量）
3. 只有 **503** 才显示「音频暂不可用，请重试」
4. 期望体验：点播放 → 转圈（最多几秒）→ 自动开播，全程无错误弹窗

## ④ 句级文字高亮跟随（可选增强，非必做）

- **可做**：按字符占比估算句子时间点（句 i 的开始时间 ≈ 前面句子字符数/总字符数 × duration），播放时高亮当前句。近似但观感不错。
- **不做**：词级精确高亮——TTS 无 timestamps，需额外对齐基础设施，不值。
- 优先级低于 ①②③。

## 验收

- 深度模块/问答/作者介绍均可点播放：首次转圈数秒后开播；再点秒播
- 同作者另一件作品点作者介绍 → **秒播**（共享验证）
- 409 场景（快速连点）不再弹错误，自动等到开播
- 进度条/剩余时间正常；语速变化时剩余时间正确
- 内容未生成的语言点播放 → 「内容生成后可听」提示（404）
