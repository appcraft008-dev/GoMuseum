# 前端交接:作者卡固定展示顺序

用户本意:作者介绍开头是**固定结构内容**,而非以 bio 叙事打头。

## 要求
`GuideArtistCard` 按**固定顺序**展示 `content.artist` 的字段:
1. **姓名** `name`(一定有)
2. **生卒年** `birth`–`death`(如 1819–1877;任一缺则只显有的)
3. **国籍/籍贯** `nationality`
4. **主要作品** `notable_works`(列表;空则不显)
5. **生平** `bio`(叙事,放最后;缺则不显)

结构化字段(1-4)在前、醒目;bio 在后。缺字段不显该行(name 必有)。

## 数据状态(后端已就绪)
`artist` 对象字段已全返(见 `content` 契约端点4)。**无后端改动**,纯前端展示顺序调整。
- 已知:`birth/death` 为**年**(非年月);`notable_works` 偶有上游 Wikidata 标签瑕疵(如 "The Fif")+ 可能含作品自身——展示即可,后端不额外清洗(v1)。
