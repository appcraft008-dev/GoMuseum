### 已敲定的呈现规则（#11 藏品图，前端实施时遵此）

- 顶部 = **官方藏品图**（`images[0]`），**不是用户照片**（照片仅识别确认步临时比对、之后丢弃）。
- **折叠式 hero**：图初见醒目，下滑读讲解时收缩成顶部细条 → 还空间给 tab；**点图全屏 + 缩放**（看细节刚需）。
  ⚠️ **中等高度(约屏幕 1/3)、快速折叠，不要全屏大图**——讲解是主角，hero 折叠让讲解尽快可读。
- **多图左右滑**（v1 一张 P18；契约 `images` 已是数组，多图时自动出轮播、前端零改）。
- 图带 `credit` 署名（合规）。

### 已敲定的呈现规则（#10 硬事实，前端实施时遵此）

- **两层呈现**：① **墙签速览**（标题下、始终可见）；② **"作品信息"可展开全表**（默认收起，不限行数）；叙事讲解 tab 仍为主体。
- **墙签 ≤3 行保证**（实测中文/英文/德文 essentials 约 2–3 行）：靠两条——① 墙签只放固定短字段子集（作者 · 年代 · 媒材缩写 · 尺寸；馆名可省，用户已在馆内）；② 啰嗦值截断（媒材过长加…）。
- **墙签是"单行流式、无字段标签"**（`作者 · 年代 · 媒材 · 尺寸`，中间 · 分隔，≤2 行）——**不是 2 列带标签的网格**
  （值自明，不需"作者:""年代:"标签）。⚠️ Claude Design 首版做成了带标签网格(~5 行)+ 作品信息默认展开 → 过厚、
  把讲解挤到屏幕底；**纠正：墙签单行流式无标签 + "作品信息"默认折叠 + 讲解 tab 尽量首屏可见**（讲解是主角、硬事实是配角）。
- **长且不定长的硬事实**（来源流转/展览史/参考文献）**只进可展开全表（默认收起），绝不进墙签**。
- **契约**：`facts` 返全量；"哪些进墙签/哪些进全表/怎么截断"是**纯前端决定**，后端零额外改动。

### 已敲定的呈现规则（#19 对话式追问 UI，前端实施时遵此）

- **底部停靠输入条 →（点击/选 chip）上拉展开聊天底部面板**（可拖高度、可收回）；收回即返回详情页阅读。
- **收起态**：详情页底部常驻 = 推荐问题 chips + "问问这幅画"输入 + 🎤。
- **展开态**：顶部锚定作品缩略图（知道在聊哪件）+ 收回箭头；**气泡式多轮**（用户右/助手左）；每条答案带
  **🔊 可选朗读** + 小字来源（"据资料/Wikipedia"）；**答完给后续推荐 chips**；**流式生成指示**；底部输入条（文本+🎤+发送）。
- **诚实"资料未涵盖"答案照常显示**；轮数将满温和提示"开个新话题"。
- **离开页/收回面板 → 停止朗读**（stop-on-leave）。
- 纯前端、契约已定（`POST /content/ask` 多轮 + `object_content.suggested_questions` + 答案朗读复用 ad-hoc `/tts/generate`）。
