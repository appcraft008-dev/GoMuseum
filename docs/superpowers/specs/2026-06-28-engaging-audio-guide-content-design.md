# 引人入胜的语音导览内容 — 设计

> **状态**:设计定稿（2026-06-28 brainstorm）。下一步 writing-plans。
> **关系**:改造 `2026-06-17-artwork-content-enrichment-mechanism-design.md` 的生成目标与质量闸语义（接地原则不变，闸的判定方式变）。

## 1. 背景与问题

当前富化讲解"完整、正确、流畅",但**像百科摘要、不吸引人、不适合语音导览**,且每段过短(中文几十~100+字)。根因不是 grounding 本身,而是**三个机制都在奖励"信息完整",没人奖励"好听好记"**:

1. **接地闸是逐句蕴含删除**:任何"材料不直接支持"的句子被删——恰好删掉导览的灵魂(第二人称引导、框架钩子、感官指引),只剩百科式事实句;**同时也是变短的元凶**。
2. **生成 prompt 目标是"分段接地讲解"**,不是"语音导览口吻"。
3. **gpt-4o-mini** 天生简短保守。

**目标**:重设生成目标与质量标准,让讲解**接地的同时引人入胜、可听、能记住一个重点**;**对所有语言有效**(目标客户主要是欧洲人,en 轴承载吸引力)。

## 2. 接地包络(最高原则,不变)

吸引力来自"怎么讲"和"讲哪个",**不是编新事实**。把句子分三类、区别对待:

- **事实主张**(可证伪:谁/何时/材质/事件/归属/画的是什么)→ **必须被材料支持,否则删**。
- **引导/讲述句**(框架、第二人称、反问、过渡、感官指引)→ **留**(非事实主张)。
- **轻度解读句**(主观感受,如"笔触显得焦躁")→ **留**,除非伪装成事实或与材料矛盾 → 删。
- **存疑偏保守**:拿不准是否事实主张时,按事实主张要求支持(守住"不脑补"铁律)。

## 3. 声音规范:叙事史口吻(当年明月原理,语言中立)

范例原理 = 当年明月《明朝那些事儿》:把枯燥史料讲得引人入胜而不胡编。**取其原理与手法,不取中文 idiom**(目标欧洲市场,en 轴承载)。操作化为可写进 prompt 的指令:

- **现代口语,像跟人讲,不端着**(去百科腔)。
- **把人当活人写**:作者/委托人有动机、处境、性格,不是简历。
- **设悬念留钩子**:"但事情没这么简单……"。
- **适度幽默/反差**,点到为止。
- **用现代类比**解释陌生概念。
- **始终接地**:每个事实有据,只是讲得好玩。

**调整(他能整篇我们不能照搬)**:① 单件 6 段短文,取声音和单段张力,不取长线结构;② 点评保持**轻**(俏皮话讲接地的反差 ✓;无据最高级 ✗);③ 料薄的件**诚实写短**,不为风格硬凑(守宁缺毋滥)。

## 4. 各段"导览剧本"(角色 + 变长)

保持现有 section code 不变(不碰契约/前端),重定义每段角色和长度。以 painting 为例(其它类目同理映射):

| 段 code | 角色 | 长度(中文字) | 要点 |
|---|---|---|---|
| overview | 钩子 | 短 60-100 | 一句抓人的开场,让你想抬头看;不要流水账 |
| artist | 有故事的人 | 中 120-180 | 与这件作品相关的一个记忆点,非简历 |
| background | 故事 | 长 180-280 | 委托/丑闻/那个时刻,叙事推进 |
| analysis | 引导看 | 长 180-280 | "注意右下角…"、构图、技法;感官指引+轻度解读集中在这 |
| significance | 带走的一个点 | 短 80-140 | 为什么重要/记住什么 = 记忆点 |
| facts | 一个趣闻 | 中 100-160 | 硬事实已在 facts 面板,本段=难忘小故事/冷知识(label 后续可改"趣闻") |

长度是**目标非硬限**;料薄时短于目标可接受。各段角色+长度以配置承载(`category_config` 加 `SECTION_ROLES`),prompt 读取。

## 5. 组件改动

**A. 生成 prompt(`prompts.build_generation_prompt`)重写**
system 从"分段接地"→"导览导演":第3节声音规范 + 第4节各段角色/长度 + 每段一个钩子一个记忆点 + 选最吸引人的可接地角度(不摊平) + 明确许可引导/解读句、禁止编造可证伪事实。user 仍是 material + 段落清单 + JSON 输出。

**B. 接地闸(`prompts.build_entailment_prompt` → 三类判定;`quality.py` 几乎不动)**
闸 prompt 改为逐句"先分类再处置"(第2节规则),输出仍是**逐句 keep/remove 布尔列表**(判官内部应用三类规则)。`quality.check_section` 代码基本不变:keep 的句拼 body;**`grounding_ratio` 语义变为"存活率"**(引导/解读句计入 keep,故自然高;低存活率=大量无据事实主张被删=料薄/脑补→needs_review)。阈值保留(可下调到 ~0.5,因语义已变)。**最小代码改动、改动主要在 prompt**。

**C. 模型**:先 `gpt-4o-mini`(`model` 参数已贯通)。日后一行切 `gpt-4o` 跑生成那遍(接地校验仍用 mini)。

**D. 翻译(`translator` prompt)**:加"保留口吻/俏皮/钩子,不直译段子"的指令,使 fr/zh 译文不丢吸引力。

## 6. 非目标(本期不做)

- **吸引力评审重写环**(方案C:LLM 按"愿意听/看懂/记住一个重点"打分+重写循环)→ 留作"mini+新 prompt 出来还平"时再加。
- **原生 zh 创作**(zh 不靠翻译、按当年明月原生写)→ 非主市场,搁置。
- **section 结构重构**(改段数/段义)→ 不碰契约;facts 段改"趣闻"label 是可选小跟进。
- **强模型默认**→ 先 mini,实测后决定。

## 7. 契约 / 迁移

无契约变更、无 Alembic 迁移。改的是**生成内容的质地**,字段不变(仍 tabs[].body)。老 App 无感。

## 8. 测试

- **闸单测(注入式 complete)**:假事实句→被删;引导句("注意右下角")→留;轻度解读句→留;解读伪装成事实/与材料矛盾→删;全段无据事实主张多→存活率低→needs_review。
- **prompt 结构测**:build_generation_prompt 含各段角色/长度;build 三类闸 prompt 含三类规则与"存疑偏保守"。
- **集成**:generate_object 端到端仍通(注入 fake 组件)。

## 9. 上线验证

1. 用新 prompt+闸(mini)**重生成 prod 14 件**(三语)。
2. 按三标准(愿意听/看懂/记住一个重点)肉眼对比 before/after + 看长度是否到位。
3. 满意 → 决定是否上 gpt-4o(实测文笔差距)+ 扩 TOP-N。不满意 → 调 prompt 或加方案C评审环。
