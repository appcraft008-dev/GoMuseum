# 跨语言翻译质量检查点清单

> 加新语言（尤其非拉丁文字：ja / ko / zh-hant）前后的验收清单。
> 所有修复机制**语言无关**（`{lang}` 占位符 / 闸信号触发），加语言=加配置，不硬编语言名单。
> 来源：2026-07-05 zh 翻译质量攻坚（从"severed head 漏翻"到"四通路标题一致"）。

## 按文字系统(script)分类，不是按地理

**决定风险的是文字系统，不是语言的地理归属。** 判定：该语言的正文主要用拉丁字母还是别的字形。

- **拉丁字母**（问题轻微，实测 0，同我们现有欧洲语言）：
  fr/de/es/it/pl、**北欧全部**（瑞典/丹麦/挪威/芬兰/冰岛，只是多变音符 å ø æ）、
  **东欧拉丁族**（捷克/斯洛伐克/匈牙利/罗马尼亚/斯洛文尼亚/克罗地亚/立陶宛/拉脱维亚/爱沙尼亚）。
  同源词多、标题保留原文/近似形式，天然少分叉。

- **非拉丁字母**（**重点**，下面每条都要跑）：
  - **东亚**：zh/ja/ko/zh-hant——最严重（无同源词、训练数据相对少）。
  - **西里尔**：俄 ru / 乌 uk / 保 bg / 塞 sr / 白俄 / 马其顿。
  - **希腊**：el。
  西里尔/希腊**比 CJK 轻一档**（艺术/技术词经希腊-拉丁词根与英语共享、LLM 语料更足），
  但检查点 ①残片 / ②专名一致 / ③标题四通路 / ④显示名质量 **全部适用**——转写进另一套字形必然分叉。
  共性症状：残片（句中夹未翻英文）、标题多通路分叉、音译不标准/不一致。

- **特例·双字母语言**（检查点⑥的直接翻版）：
  **塞尔维亚语 sr** 同时用西里尔和拉丁两套（digraphia），Wikidata 标签可能任一套——
  和 zh 简繁同理，加 sr 时要定"统一到哪套字形"（同 zh-hans/OpenCC 的处理）。

## 六个检查点（每条：查什么 / 命令 / 修复机制）

### ① 源语言残片（普通词未翻译）
- **症状**：非拉丁译文里夹未翻译英文普通词（"约翰的 severed head"）。
- **查**：正文/问答里 `(?<![A-Za-z])[a-z]{3,}` 小写拉丁词串（排除专有名词）。西里尔/希腊译文里夹的英文同样是小写拉丁串，一样被抓——**检测按文字类通用**。
- **谁要查**：只对**非拉丁语言集**跑（拉丁字母语言正文本就含拉丁字母，查了全是误报）。集合当前 `{zh}`；**加西里尔/希腊时把 `ru/uk/bg/sr/el` 等加进这个集合即可——是加配置，不是加代码分支**（`ponytail:` 集合够用，大到不好维护再上 Unicode 脚本检测）。
- **修**（已实现，语言无关）：翻译 prompt 规则(1b)"全部译出不留残片" + 忠实度闸也判残片 + 闸失败→gpt-4o 兜底重译。
- **⚠️ 扫描范围**：必须扫**全部已发布段**（guide + 深度模块 background/analysis/facts + QA）——只扫 guide+QA 会漏深度模块里的残片（2026-07-06 教训:zh-hant "revelry" 藏在 background 段被漏）。
- **⚠️ 残片命中要人工分辨专有名词语境**：机构/作品官方名（如 "Salon des artistes français"、括号内官方原名）**本该保留**，是误报；真残片是句中夹的普通词（revelry/severed head/nude）。闸和 gpt-4o 会正确保留专有名词、只该译普通词。

### ② 专有名词一致 + 标准译名
- **症状**：同一人名/地名全文渲染不一致（一处 Salome 一处莎乐美）；音译非标准（奥林比亚 vs 奥林匹亚）。
- **查**：同一实体多处渲染是否一致；音译是否标准。
- **修**：翻译 prompt 规则(2)"有既定 `{lang}` 译名则用之 + 全文一致"。

### ③ 标题多通路一致（本次新增，最隐蔽）
- **症状**：同一藏品标题在**显示名 / 讲解 / 深度模块 / 问答**四处不一（显现 vs 幻影）。根因：四条独立翻译通路各译各的。
- **查**：各通路 `《标题》` == `title_i18n[lang]`（**排除引用其他作品**——那些本就该不同）。
- **修**：glossary——把 `title_i18n[lang]` 作为规范标题注入内容翻译 prompt，正文引用一律用它。已穿到 guide/deep/qa 三条通路（`translate_object(titles=)` / `suggest(titles=)`）。

### ④ 显示名质量（title_i18n 本身）
- **症状**：显示名标题自己有残片（"卧 nude 女人"）或非标准音译。显示名翻译（`translate_name`）比正文弱。
- **查**：`title_i18n[lang]` 有无拉丁残片 / 非标准音译。
- **修**：`translate_name` 用 gpt-4o + prompt 硬化（无残片/标准音译）；存量用 `onboard names --retranslate-langs <lang>` 重译无权威标签的机翻显示名。

### ⑤ 忠实度（不增删事实）
- **查**：忠实度闸（`check_faithfulness`）已覆盖。
- **修**：闸失败→needs_review 不发布（宁缺毋滥）；闸失败→gpt-4o 重译。

### ⑥ 字形规范（zh 类）
- **症状**：简繁混杂（愛德華·馬奈）。
- **查**：目标字形外的字符残留（zh 查繁体字）。
- **修**：`zh-hans > zh-cn > zh` 标签优先 + OpenCC t2s 兜底；存量 `names --refresh-langs zh`。
- **注**：zh-hant 作为独立语言时，方向相反（保留繁体，不转简）——加 zh-hant 时需确认 OpenCC 方向按 target 区分。
- **西里尔类推**：塞尔维亚语 sr 的西里尔/拉丁 digraphia 同理——定统一字形，抓另一套字形的残留。

### ⑦ 整段语言纯度（核心差异点，2026-07-10 加）
- **症状**：整段/整字段是**别的语言**（作者 bio 出法语、zh 问答混整句英文、en 轴心镜像法语源）。①只抓"残片"，⑦抓"整段跑偏"。
- **查**：`lang_detect.text_in_language(text, lang)`——非拉丁按目标字形占比、拉丁用 lingua。全库重扫：`python scripts/rescan_language.py orsay --target <env>`（找出并重译污染段，幂等）。
- **修**（已实现，语言无关）：翻译/生成路径接语言一致性闸——不符→gpt-4o 重译→仍不符 needs_review 不发布。bio 用检测器统一判定（取代查中/法语打地鼠）。
- **语言无关·未来语言自动覆盖**：检测候选集派生自 `DEFAULT_LANGUAGES`；加语言即纳入。加非拉丁语言时把它加进 `lang_detect._NONLATIN_SCRIPT`（一行）；拉丁语言若 lingua 支持则自动。
- **加语言验收必查**：新语言下抽查各内容（讲解/深度/问答/bio）无整段外语；跑 `rescan_language` 应零污染。

## 加一种非拉丁语言的标准动作

1. 配置：契约"加语言 checklist"五步（DEFAULT_LANGUAGES/静态标签表/前端 ARB…）。
2. 铺显示名：`onboard names --target <env> --langs <lang>`（权威标签→gpt-4o 翻译）。
3. 补内容：`onboard translate --target <env> --langs <lang>`（存量）/ 懒翻译（按需）。
4. **跑本清单①–⑥扫描**，确认各项归零（残留先分辨是否"引用其他作品"的误报）。
5. 有缺陷：`--retranslate-langs`（显示名）/ 重译内容（跟随修好的显示名）。

## 语言无关机制清单（复用，勿为单一语言硬编码）

- 忠实度闸 + gpt-4o 兜底重译（闸信号触发，非语言名单）
- glossary：`title_i18n[lang]` 穿三条翻译通路
- `translate_name` gpt-4o + 硬化 prompt
- 命令：`names --refresh-langs` / `--retranslate-langs` / `translate --langs`
